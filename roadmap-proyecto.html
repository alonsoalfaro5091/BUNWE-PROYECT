¿Qué es JSON? — El idioma común de tu app
📦 JSON = paquete de datos entre Flutter y Python
Como un diccionario Python que viaja por la red
Así se ve tu JSON de sueño
{
  "puntuacion": 85,
  "horas": 7.5,
  "racha_dias": 5,
  "ultima_noche": "2025-05-28",
  "historial": [
    { "dia": "Lun", "horas": 7.0 },
    { "dia": "Mar", "horas": 6.5 },
    { "dia": "Mie", "horas": 8.0 }
  ]
}
Cómo entenderlo sin complicarse
→Es idéntico a un diccionario Python: clave + valor
→Python lo genera y lo manda. Flutter lo recibe y pinta la UI
→Es texto plano: puedes verlo en el navegador escribiendo la IP de la Pi
→{ } = un objeto (una fila). [ ] = una lista de objetos
→No hay nada que instalar para entenderlo, se lee solo

Raspberry Pi OS + SSH headless
Sin monitor ni teclado · desde Windows, Mac o Linux
1Descargar Raspberry Pi Imager
Descarga el programa oficial desde raspberrypi.com/software. Gratuito para Windows, Mac y Linux. Inserta la microSD de la Pi en tu PC antes de continuar.
💡 Usa Raspberry Pi OS Lite (64-bit) — sin escritorio. Al no tener interfaz gráfica consume menos RAM y CPU, dejando más recursos para FastAPI y MySQL.
2Configurar antes de grabar — clave del headless
En el Imager, antes de grabar, haz clic en el ícono ⚙️ o presiona Ctrl+Shift+X. Esto es lo que configuras:
Opciones avanzadas del Imager — qué hace cada una
✓Habilitar SSH → Usar autenticación por contraseña
# Sin esto, la Pi bloquea todas las conexiones remotas por seguridad.
# SSH es el "cable virtual" que conecta tu PC con la Pi sin monitor.
 
✓Usuario: pi Contraseña: (la que elijas)
# Son las credenciales para entrar a la Pi por SSH.
# Guárdalas bien — si las olvidás tenés que grabar la SD de nuevo.
 
✓WiFi → SSID: (nombre de tu red) Password: (clave WiFi)
# Le dice a la Pi a qué red conectarse al encender por primera vez.
# Sin esto la Pi arranca sin red y no podés encontrarla desde tu PC.
 
✓Hostname: raspberrypi.local
# Es el "nombre de dominio" de la Pi en tu red local.
# Con este nombre podés conectarte sin saber la IP todavía.
⚠️ Si no configurás el WiFi aquí, la Pi no se conectará sola. No hay segunda oportunidad sin volver a grabar la SD.
3Primer arranque y conexión SSH
Graba la imagen, insertá la SD en la Pi y encendela. Esperá 2 minutos. Luego desde tu PC:
Terminal de tu PC (Windows / Mac / Linux)
$ping raspberrypi.local
# Envía un "hola" a la Pi y espera respuesta.
# Si responde con tiempos en ms → la Pi está viva en la red.
# Si no responde → esperá 1 minuto más y volvé a intentar.
 
$ssh pi@raspberrypi.local
# "ssh" = Secure Shell, protocolo de conexión remota cifrado.
# "pi" = tu nombre de usuario en la Pi.
# "raspberrypi.local" = el hostname que le diste en el Imager.
# A partir de este momento cada comando que escribás
# se ejecuta DENTRO de la Pi, no en tu PC.
# Es como teletransportarte a la Raspberry sin necesitar monitor.
✓ Cuando veas pi@raspberrypi:~$ estás adentro. Todo lo que escribas ahora ocurre en la Pi.
4Actualizar el sistema
Siempre lo primero al entrar por primera vez. Sin esto podés instalar versiones viejas o recibir errores raros.
Dentro de la Pi (vía SSH)
$sudo apt update
# "sudo" = ejecutar como administrador (superusuario del sistema).
# Sin "sudo" el sistema te negaría el permiso para modificar paquetes.
# "apt" = el gestor de paquetes de Linux (el equivalente a una App Store).
# "update" = descarga el catálogo actualizado de qué está disponible.
# ⚠️ No instala nada todavía, solo actualiza la lista interna.
 
$sudo apt upgrade -y
# "upgrade" = descarga e instala TODAS las actualizaciones disponibles.
# Incluye parches de seguridad y versiones nuevas de Python, etc.
# "-y" = responde "sí" automáticamente a todas las confirmaciones.
# Puede tardar 3-8 minutos. Verás texto pasando — es normal.
5Ver IP y asignar IP fija
Sin IP fija, el router puede cambiarle la dirección a la Pi cada vez que se reinicia y Flutter perdería la conexión sin razón aparente.
Ver la IP actual y la MAC de la Pi
$hostname -I
# Muestra la IP actual de la Pi, ej: 192.168.1.105
# Esta es la dirección que pondrás en Flutter como baseUrl.
# Anotala — la usarás para probar antes de tener Tailscale.
 
$ip link show wlan0
# Muestra la dirección MAC de la interfaz WiFi (wlan0).
# Ejemplo de salida: "link/ether aa:bb:cc:dd:ee:ff"
# La MAC es el identificador único del hardware de la Pi.
# Entrá al panel de tu router (192.168.1.1), buscá esa MAC
# en la sección DHCP y asignale siempre la misma IP fija.
💡 Asignar la IP fija se hace desde el router, no desde la Pi. Buscá en el panel de tu router "Reserva DHCP" o "IP estática por MAC".

MySQL (MariaDB) en Raspberry Pi
MariaDB es 100% compatible con MySQL · se instala con un comando
1Instalar MariaDB
MariaDB es la versión de MySQL que viene en los repositorios oficiales de Raspberry Pi OS. Es idéntica en sintaxis SQL.
Dentro de la Pi (vía SSH)
$sudo apt update
# Refresca el catálogo de paquetes. Siempre antes de instalar algo.
# Sin esto podés instalar una versión vieja que ya fue reemplazada.
 
$sudo apt install mariadb-server -y
# "install mariadb-server" = descarga e instala el motor de base de datos.
# "-y" = confirma automáticamente sin pedir aprobación.
# Después de esto, MariaDB ya está corriendo en segundo plano.
 
$sudo systemctl status mariadb
# Verifica que el servicio esté activo.
# Debés ver "Active: active (running)" en verde.
# Si dice "inactive" ejecutá: sudo systemctl start mariadb
2Asegurar la instalación — comando moderno de MariaDB
Por defecto MariaDB queda sin contraseña, lo que es un riesgo de seguridad. El comando para asegurarla cambió en versiones recientes.
⚠️ Usar este comando — NO el antiguo mysql_secure_installation
$sudo mariadb-secure-installation
# En versiones nuevas de MariaDB en Pi OS 64-bit, las herramientas
# se renombraron de "mysql_*" a "mariadb-*" para evitar confusiones.
# Si escribís "mysql_secure_installation" vas a recibir un error.
El asistente hace 7 preguntas. Respuestas recomendadas:
Respuestas dentro del asistente interactivo
1.Enter current password for root:
# → Solo presioná Enter (vacío). Todavía no hay contraseña configurada.
 
2.Switch to unix_socket authentication?
# → Escribí n (No). Queremos autenticación por contraseña clásica,
# no por el usuario del sistema operativo. Python la necesita así.
 
3.Change the root password?
# → Escribí y (Sí) y luego tu contraseña segura.
# ⚠️ Al escribir la contraseña NO verás nada en pantalla (ni asteriscos).
# Es normal, es una medida de seguridad de Linux. Escribí y Enter.
 
4.Remove anonymous users?
# → Escribí y (Sí). Elimina usuarios de prueba que vienen por defecto
# y que cualquiera podría usar para acceder sin contraseña.
 
5.Disallow root login remotely?
# → Escribí y (Sí). Impide que alguien intente hackear el usuario
# "root" desde fuera de la Pi por la red. Root solo entra localmente.
 
6.Remove test database and access to it?
# → Escribí y (Sí). Borra una base de datos vacía de ejemplo
# que viene pre-instalada y no sirve para nada en producción.
 
7.Reload privilege tables now?
# → Escribí y (Sí). Aplica todos los cambios de permisos de inmediato.
# Sin esto los cambios no toman efecto hasta reiniciar el servicio.
✓ Al terminar verás "Thanks for using MariaDB!" — base de datos blindada y lista.
⚠️ Guardá la contraseña de root en un lugar seguro. La vas a necesitar para conectar Python a la BD.
3Crear la base de datos y las tablas
Entrar al CLI de MariaDB
$sudo mariadb -u root -p
# "mariadb" = abre el cliente de línea de comandos de la base de datos.
# "-u root" = conectarse como el usuario administrador "root".
# "-p" = pedir contraseña al conectar (la que creaste en el paso 2).
# Al entrar verás el prompt "MariaDB [(none)]>" — ya estás adentro.
Crear BD, usuario y tablas (dentro del CLI de MariaDB)
>CREATE DATABASE sleep_app;
# Crea la base de datos del proyecto. El nombre es libre,
# pero acordate de usarlo igual en Python más adelante.
 
>CREATE USER 'appuser'@'localhost' IDENTIFIED BY 'tu_password';
# Crea un usuario específico para la app (NO uses root en Python).
# 'appuser' = nombre del usuario para FastAPI.
# '@localhost' = solo puede conectarse desde la misma Pi, más seguro.
# 'tu_password' = contraseña que usará Python para conectarse.
 
>GRANT ALL ON sleep_app.* TO 'appuser'@'localhost';
# Otorga todos los permisos sobre sleep_app al usuario appuser.
# Sin esto appuser no puede leer ni escribir tablas, FastAPI fallaría.
 
>FLUSH PRIVILEGES;
# Recarga la tabla de permisos. Hace efectivos los GRANT anteriores.
# Sin esto el sistema no "ve" los permisos que acabás de dar.
 
>USE sleep_app;
# Selecciona la BD sleep_app como activa para los siguientes comandos.
# Verás que el prompt cambia a "MariaDB [sleep_app]>".
 
>CREATE TABLE registro_sueno (
  id INT AUTO_INCREMENT PRIMARY KEY,
  # id: clave única que se incrementa sola. Identifica cada registro.
  fecha DATE NOT NULL,
  # fecha: el día de la noche de sueño. NOT NULL = obligatorio.
  horas DECIMAL(4,1) NOT NULL,
  # horas: permite decimales como 7.5. DECIMAL(4,1) = hasta 999.9
  puntuacion INT DEFAULT 0,
  # puntuacion: 0 a 100. DEFAULT 0 si no se envía desde Flutter.
  notas TEXT
  # notas: campo libre, opcional. TEXT permite texto largo.
>);
>EXIT;
# Cierra la sesión del CLI y vuelve al terminal normal de la Pi.
4Insertar datos de prueba
Entrar con el usuario appuser e insertar datos
$mariadb -u appuser -p sleep_app
# Conectarse como appuser (el usuario que usará Python/FastAPI).
# Buena práctica: probar siempre con el mismo usuario que usará la app.
 
>INSERT INTO registro_sueno (fecha, horas, puntuacion) VALUES
('2025-05-22', 7.0, 78), ('2025-05-23', 6.5, 70),
('2025-05-24', 8.0, 90), ('2025-05-25', 7.5, 85),
('2025-05-26', 6.0, 65), ('2025-05-27', 7.0, 80),
('2025-05-28', 7.5, 85);
# INSERT = agrega filas a la tabla.
# Estos 7 días simulan una semana de historial real para
# que Flutter tenga algo que mostrar desde el primer día de prueba.
✓ Ya tenés 7 días de datos reales. Flutter los verá cuando conectes la app en la Semana 3.

FastAPI + Uvicorn en Raspberry Pi
El servidor que responde JSON a Flutter · Python puro
1Instalar dependencias
Dentro de la Pi (vía SSH)
$sudo apt install python3-pip python3-venv -y
# "python3-pip" = el gestor de paquetes de Python (instala librerías).
# "python3-venv" = permite crear entornos virtuales de Python.
 
$mkdir ~/sleep-api && cd ~/sleep-api
# "mkdir" = crea una carpeta llamada "sleep-api" en el home de la Pi.
# "&&" = ejecuta el siguiente comando solo si el anterior tuvo éxito.
# "cd" = entra a esa carpeta. Todos los archivos del proyecto van aquí.
 
$python3 -m venv venv
# Crea un entorno virtual llamado "venv" dentro del proyecto.
# Un entorno virtual aísla las librerías de este proyecto del sistema.
# Así FastAPI no interfiere con otros programas de Python en la Pi.
 
$source venv/bin/activate
# "Activa" el entorno virtual. El prompt cambia a "(venv) pi@raspberrypi"
# Eso significa que todo pip install va ahora al entorno, no al sistema.
# ⚠️ Debés activarlo CADA VEZ que abras una nueva sesión SSH.
 
$pip install "fastapi[standard]" PyMySQL
# "fastapi[standard]" = instala FastAPI + Uvicorn + extras recomendados.
# "PyMySQL" = conector Python para hablar con MariaDB/MySQL.
# Sin PyMySQL, Python no puede conectarse a la base de datos.
2Crear el archivo main.py
Crear y editar el archivo con nano
$nano ~/sleep-api/main.py
# "nano" = editor de texto en la terminal. Abre el archivo para editarlo.
# Para guardar: Ctrl+O → Enter. Para salir: Ctrl+X.
Contenido de main.py — con explicación de cada parte
from fastapi import FastAPI
# Importa la clase principal de FastAPI para crear la app.
from fastapi.middleware.cors import CORSMiddleware
# CORS permite que Flutter (otro dominio/IP) haga peticiones al server.
# Sin esto el servidor bloquearía las peticiones de la app.
import pymysql
# El conector que instalamos. Permite que Python hable con MariaDB.
 
app = FastAPI()
# Crea la instancia principal de la aplicación. Todo se registra aquí.
app.add_middleware(CORSMiddleware, allow_origins=["*"])
# allow_origins=["*"] = acepta peticiones desde cualquier IP.
# Ideal para desarrollo. En producción limitarías a IPs específicas.
 
def get_conn():
# Función que abre una conexión a la base de datos.
# Se llama cada vez que un endpoint necesita consultar datos.
  return pymysql.connect(host="localhost",
  # "localhost" = la BD está en la misma Pi que FastAPI.
    user="appuser", password="tu_password",
    # Usar appuser, NUNCA root en la app. Principio de mínimo privilegio.
    database="sleep_app", cursorclass=pymysql.cursors.DictCursor)
    # DictCursor devuelve las filas como diccionarios Python,
    # que FastAPI convierte automáticamente a JSON para Flutter.
 
@app.get("/historial")
# "@app.get" = registra esta función como endpoint HTTP GET.
# Cuando Flutter haga GET a /historial, se ejecuta la función abajo.
def historial():
  conn = get_conn()
  with conn.cursor() as cur:
  # "with" cierra el cursor automáticamente al terminar. Buena práctica.
    cur.execute("SELECT * FROM registro_sueno ORDER BY fecha DESC LIMIT 7")
    # Trae los últimos 7 días ordenados del más reciente al más viejo.
    rows = cur.fetchall()
    # "fetchall" guarda todos los resultados en la variable "rows".
  conn.close()
  # Cierra la conexión. Importante para no agotar las conexiones disponibles.
  return {"historial": rows}
  # FastAPI convierte este dict Python a JSON automáticamente.
  # Flutter recibirá: {"historial": [{...}, {...}, ...]}
3Arrancar el servidor
Lanzar FastAPI
$cd ~/sleep-api && source venv/bin/activate
# Entrá a la carpeta del proyecto y activá el entorno virtual.
# Si no activás el venv, Python no encontrará FastAPI ni PyMySQL.
 
$uvicorn main:app --host 0.0.0.0 --port 8000 --reload
# "uvicorn" = el servidor web que corre la app FastAPI.
# "main:app" = archivo main.py, objeto "app" dentro de él.
# "--host 0.0.0.0" = acepta conexiones de CUALQUIER IP de la red.
# ⚠️ Sin este flag, solo la Pi misma puede acceder. Flutter no llegaría.
# "--port 8000" = el puerto donde escucha. Como el canal de TV del server.
# "--reload" = reinicia automáticamente si modificás el código.
# Solo usar --reload en desarrollo, quitarlo en producción.
✓ Abrí el navegador en http://IP-DE-LA-PI:8000/historial y verás el JSON con los datos de MySQL.
💡 FastAPI genera documentación interactiva automáticamente en http://IP-DE-LA-PI:8000/docs — podés probar todos los endpoints desde el navegador sin usar Flutter todavía.
4Autoarranque al encender la Pi — servicio systemd
Para que FastAPI arranque solo cada vez que la Pi se enciende, sin necesidad de entrar por SSH.
Crear el archivo de servicio
$sudo nano /etc/systemd/system/sleep-api.service
# Crea un archivo de configuración en la carpeta de servicios del sistema.
# "systemd" es el gestor de servicios de Linux — controla qué
# programas arrancan automáticamente al encender el sistema.
Contenido del archivo .service
[Unit]
Description=Sleep App FastAPI Server
# Descripción legible del servicio. Aparece en los logs del sistema.
After=network.target
# Arranca DESPUÉS de que la red esté lista. Sin esto podría arrancar
# antes de tener WiFi y fallar al intentar escuchar en la red.
 
[Service]
User=pi
# El servicio corre como el usuario "pi", no como root. Más seguro.
WorkingDirectory=/home/pi/sleep-api
# La carpeta desde donde se ejecuta uvicorn. Debe existir.
ExecStart=/home/pi/sleep-api/venv/bin/uvicorn main:app --host 0.0.0.0 --port 8000
# Ruta absoluta al uvicorn del entorno virtual. Importante: sin --reload
# en producción, porque --reload consume más recursos.
Restart=always
# Si FastAPI se cae por algún error, systemd lo reinicia automáticamente.
# Hace que el servidor sea mucho más confiable sin intervención manual.
 
[Install]
WantedBy=multi-user.target
# Define en qué estado del arranque del sistema se activa este servicio.
# "multi-user" = cuando el sistema ya está listo para múltiples usuarios.
Activar y arrancar el servicio
$sudo systemctl daemon-reload
# Le dice a systemd que recargue la lista de servicios.
# Necesario siempre que creás o modificás un archivo .service.
 
$sudo systemctl enable sleep-api
# "enable" = marca el servicio para que arranque automáticamente al encender.
# Crea un enlace simbólico en la secuencia de arranque del sistema.
 
$sudo systemctl start sleep-api
# "start" = arranca el servicio ahora mismo, sin reiniciar la Pi.
 
$sudo systemctl status sleep-api
# Muestra si el servicio está corriendo correctamente.
# Buscá "Active: active (running)" para confirmar que todo está bien.
# Si hay error, verás el mensaje exacto aquí para saber qué falló.
✓ Ahora FastAPI arranca solo cada vez que encendés la Pi. No hace falta entrar por SSH para iniciarlo.

Tailscale en Raspberry Pi
Acceso desde cualquier red · sin abrir puertos · gratis
1Crear cuenta en Tailscale
Entrar a tailscale.com y crear una cuenta gratuita con Google o GitHub. El plan gratuito personal soporta hasta 100 dispositivos.
💡 Los 4 del equipo pueden conectar sus teléfonos a la misma cuenta, o cada uno puede tener su cuenta e invitarse mutuamente desde el panel de administración.
2Instalar Tailscale en la Pi
Dentro de la Pi (vía SSH)
$curl -fsSL https://tailscale.com/install.sh | sh
# "curl" = descarga un archivo desde internet.
# "-fsSL" = fail silently, follow redirects, SSL. Flags de seguridad.
# "| sh" = pasa el contenido descargado directamente al intérprete shell.
# Este comando descarga y ejecuta el instalador oficial de Tailscale.
# Agrega el repositorio oficial y luego instala el paquete.
3Autenticar la Pi en tu red Tailscale
Conectar la Pi a la cuenta
$sudo tailscale up
# "tailscale up" = conecta la Pi a la red privada de Tailscale.
# Va a mostrarte una URL del tipo: https://login.tailscale.com/a/xxxxx
# Copiá esa URL, abríla en el navegador de tu PC e iniciá sesión.
# Eso "empareja" la Pi con tu cuenta de Tailscale permanentemente.
✓ La Pi aparece en tailscale.com/admin con una IP permanente tipo 100.x.x.x. Esa IP no cambia nunca.
4Autoarranque de Tailscale al encender
Habilitar como servicio del sistema
$sudo systemctl enable tailscaled
# "enable" = arranca Tailscale automáticamente al encender la Pi.
# Sin esto, al reiniciar la Pi deberías conectar Tailscale manualmente.
# La Pi quedaría inaccesible desde fuera de casa hasta que lo hagas.
 
$sudo systemctl start tailscaled
# Arranca el servicio ahora mismo sin necesidad de reiniciar.
5Instalar Tailscale en los 4 teléfonos Android
Buscar "Tailscale" en Google Play Store, instalar e iniciar sesión con la misma cuenta. Desde ese momento los teléfonos y la Pi están en la misma red virtual.
💡 Una vez conectados, los teléfonos llegan a la Pi usando la IP 100.x.x.x desde cualquier WiFi, datos móviles o incluso desde otro país.
6Actualizar la URL en Flutter
En el archivo de servicio HTTP de tu proyecto Dart
// Antes (solo funciona en el WiFi de casa):
const String baseUrl = "http://192.168.1.105:8000";
 
// Después (funciona desde cualquier red con Tailscale activo):
const String baseUrl = "http://100.x.x.x:8000";
// La IP 100.x.x.x es la que te dio Tailscale en el panel de admin.
// Ese es el único cambio necesario en Flutter para acceso remoto.
✓ Con este cambio la app funciona dentro y fuera de casa sin ninguna configuración adicional.
7Comandos útiles de Tailscale
Referencia rápida
$tailscale ip
# Muestra tu IP de Tailscale (la 100.x.x.x). Usala en Flutter.
 
$tailscale status
# Lista todos los dispositivos conectados a tu red Tailscale.
# Útil para verificar que los 4 teléfonos estén en la red.
 
$tailscale ping 100.x.x.x
# Prueba la conexión con otro dispositivo de la red Tailscale.
# Si responde → Flutter puede conectarse. Si no → revisar que Tailscale
# esté activo tanto en la Pi como en el teléfono.

✓ IP fija permanente (100.x.x.x)
✓ Gratis, fácil, rápido
✓ Funciona con datos móviles

  1. Cuenta en ngrok.com
  2. snap install ngrok en la Pi
  3. ngrok http 8000 → URL pública
  4. Flutter usa esa URL
✓ Listo en 5 minutos
✗ URL cambia cada vez (plan gratis)

  1. Cuenta en cloudflare.com
  2. Instalar cloudflared
  3. Crear túnel con dominio fijo
✓ Dominio fijo permanente
✗ Más configuración inicial

  1. Panel del router → abrir puerto 8000
  2. Dominio gratis en duckdns.org
  3. Script en la Pi para actualizar IP
✓ Sin depender de terceros
✗ Requiere acceso al router
